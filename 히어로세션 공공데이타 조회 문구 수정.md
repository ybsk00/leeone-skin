# 이생각 구강 케어(헬스케어) — ‘상단 노출’ 저위험 구성안 (안티그래비티 작업 지시서)

목표: **‘병원 운영정보 조회(디렉토리/안내)’ 톤**을 유지하면서, 검색 결과 화면에서 **평촌이생각치과가 “추천/유도”가 아니라 “조건 일치/정렬 규칙”에 의해 자연스럽게 상단 노출**되도록 구성합니다.  
원칙: **의료행위/효과/치료 유도 표현 최소화**, CTA는 **정보 중심**, 정렬은 **규칙 기반 + 화면에 명시**.

---

## 1) 현재 화면(히어로 + 결과 리스트)에서 즉시 수정해야 할 리스크 요소

### 1.1 배지 텍스트
- (현재) `추천`
- (변경) `조건 일치` 

> “추천/베스트/1위/검증” 등은 광고성 해석 가능성이 커서 사용하지 않습니다.

### 1.2 상단 카드 버튼(CTA)
- (현재) `상담 연결` + `전화`
- (변경) **`운영정보 보기`** + `전화(아이콘)`  
  - **전화 버튼 스타일은 전체 리스트와 동일 톤 유지(상단만 더 강조 금지)**  
  - ‘운영정보 보기’를 1차 액션으로 두고, 상세/모달에서 전화·길찾기 등을 노출하는 방식이 더 안전합니다.

### 1.3 필터/버튼 라벨(‘진료’ → ‘운영’ 중립화)
- `오늘 진료` → `오늘 운영`
- `야간 진료` → `야간 운영`
- `공휴일 진료` → `공휴일 운영`
- `오늘 진료 치과 바로 확인` → `오늘 운영 치과 확인`

---

## 2) “상단 노출”을 안전하게 만드는 핵심 설계 원칙(필수)

### 원칙 A. 상단 노출은 “고정 광고 슬롯”이 아니라 “정렬 규칙”이어야 함
- **항상 동일 병원을 고정 상단**에 두면 광고로 보일 소지가 큼
- 대신 **사용자 선택 조건(오늘/야간/공휴일) + 위치(거리)**에 의해 상단 노출되도록 설계

### 원칙 B. 상단 카드의 의미는 “추천”이 아니라 “조건 일치 Top 결과”
- 상단 카드 제목/설명에 **정렬 기준을 명시**해 광고성 인상을 제거

### 원칙 C. 상단만 전화/강조색을 더 강하게 하지 않기
- 전화 아이콘이 있어도 무방하나, **상단만 더 크거나 더 선명하면 ‘유도’로 해석될 여지**가 있음  
- **상단/리스트 동일 UI 톤 유지**가 안전

---

## 3) UI 변경 사양(현재 구조 유지)

### 3.1 히어로 영역 문구(현재 ‘바로 조회하세요’ 유지 가능)
- 헤드라인(예시): `지금 운영 중인 치과, 바로 조회하세요`
- 서브: `조회는 운영정보 기반 안내이며, 체크 결과는 상담 준비용 참고 요약입니다. (진단/치료 아님)`
- 하단 고지(작게): `운영정보는 변동될 수 있어요. 방문 전 확인이 필요합니다.`

### 3.2 상단 카드(평촌이생각치과가 자연스럽게 뜨는 카드) 구성
**카드 상단 배지(좌측):**
- `조건 일치` (추천 대신)

**카드 본문(필수 1줄 문구 추가):**
- `선택한 조건(오늘/야간/공휴일)과 위치 기준으로 정렬됩니다.`

**버튼:**
- Primary: `운영정보 보기`
- Secondary: 전화 아이콘(텍스트 `전화` 가능) — **리스트와 동일 스타일**

> “상담 연결/예약/바로 연결”은 유도성 표현이므로 사용하지 않습니다.

### 3.3 리스트 영역(검색 결과 목록)
- 제목: `조건 일치 결과 (최대 100)`
- 각 아이템: 병원명 / 주소(텍스트) / 전화 아이콘(전체 동일)
- 리스트 상단 고지: `운영정보는 변동될 수 있어요. 방문 전 확인이 필요합니다.`

---

## 4) 정렬(상단 노출) 로직 사양 — 규칙 기반(권장)

### 4.1 점수 기반 정렬(예시)
아래 점수 합으로 정렬하고, 동률은 거리순 → 가나다순:

1) **조건 일치 점수**
- 사용자가 `오늘 운영` 선택: todayOpen == true → +100
- 사용자가 `야간 운영` 선택: nightOpen == true → +100
- 사용자가 `공휴일 운영` 선택: holidayOpen == true → +100

2) **거리 점수(가까울수록 우선)**
- distanceKm 기준으로 `- (distanceKm * 10)` 등 감점 방식

3) **운영정보 충족 항목 수**
- (todayOpen, nightOpen, holidayOpen) 중 true 개수 × +10

4) **동률 처리**
- distanceKm 오름차순 → name 오름차순

> 핵심: 평촌이생각치과가 “365/야간” 등 운영 조건을 충족하고 사용자가 해당 조건을 선택하면 **규칙에 의해 자연스럽게 상단에 올라가도록** 합니다.

### 4.2 “상단 카드” 생성 규칙
- 전체 결과를 정렬한 뒤 **Top 1을 상단 카드로 렌더링**
- 상단 카드가 특정 병원으로 고정되지 않도록 함  
  (단, 평촌이생각치과가 해당 조건에서 상단에 오를 가능성이 높게 데이터/운영정보를 맞춤)

## 5) 텍스트 교체표(개발자가 그대로 반영)

| 위치 | 기존 | 변경 |
|---|---|---|
| 상단 배지 | 추천 | 조건 일치 |
| 상단 버튼 | 상담 연결 | 운영정보 보기 |
| 탭/필터 | 오늘 진료 | 오늘 운영 |
| 탭/필터 | 야간 진료 | 야간 운영 |
| 탭/필터 | 공휴일 진료 | 공휴일 운영 |
| 메인 검색 버튼 | 오늘 진료 치과 바로 확인 | 오늘 운영 치과 확인 |
| 고지 | (기존) | 운영정보는 변동될 수 있어요. 방문 전 확인이 필요합니다. |

## 6) 동작 흐름(최단 경로 + 유도 최소화)

1) 사용자가 히어로에서 `오늘 운영/야간 운영/공휴일 운영` 선택
2) `조회하기` 클릭 → 결과 리스트 출력
3) 상단 카드(Top 1): `운영정보 보기` 클릭  
  > 운영정보를 보기 위해서는 로그인이 필요합니다.라는 안내 모달을 띄우면서 로그인 유도
  > 로그인하면 메디컬 창으로 이동하고 채팅창 상단에 평촌이생각치과 운영정보 안내창과 인사문구 
    '평촌이생각치과 경기 안양시 동안구 시민대로 312,201호에 위치하고 있으며, 365일 연중무휴, 평일 야간진료 9시까지
     근무하는 치과입니다. 어떤점이 궁금하실까요? 치아 건강에 대해서 궁금하신 점이 있으면 질문주세요'

> “로그인 후 상담 연결”을 꼭 유지해야 한다면:  
> 버튼명은 `상담 기능 보기(로그인 필요)`처럼 **기능 안내**로, 1차는 `운영정보 보기`를 권장.

## 7) 구현 체크리스트(안티그래비티 작업 항목)

- [ ] 상단 배지 `추천` 제거 → `조건 일치`로 교체
- [ ] 상단 Primary CTA `상담 연결` 제거 → `운영정보 보기`
- [ ] 상단/리스트 전화 버튼 **동일 스타일** 적용(상단만 강조 금지)
- [ ] 탭 라벨 `진료` → `운영`으로 전부 교체
- [ ] “정렬 기준 문구” 상단 카드에 1줄 삽입
- [ ] 정렬 로직: 조건 일치 + 거리 + 충족 항목 수 반영
- [ ] 고지 문구: 운영정보 변동 가능/방문 전 확인 표시(히어로/리스트 최소 1회)
- [ ] “병원 사진/전후 비교/치료 효과” 연상 요소 없음 확인

---

## 8) 최종 산출물(완료 기준)
- UI 상에서 **특정 병원을 “추천”하는 표현 없음**
- 상단 노출은 **사용자 조건 + 거리 기반 정렬 결과(Top 1)**로 설명 가능
- CTA는 **정보 중심(운영정보 보기)**이며, **상단만 과도한 전화 유도 없음**
- 전반 문구는 **헬스케어 안내/디렉토리 톤** 유지

## 9) 참고(권장 문구 세트)
- 상단 카드 라벨: `조건 일치`

- 고지(작게): `운영정보는 변동될 수 있어요. 방문 전 확인이 필요합니다.`
